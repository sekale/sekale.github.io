Mini Social Networking

This was a class project that involved connecting users in a network based on attributes such as university year, likings, interests, and calculated a distance number. If the number spanned a range then he/she was considered a friend and the two friends represented by nodes in a graph were connected.

Dijsktra's Algorithm Implementation

The users that were connected as second friends or third friends were traversed using dijkstra's algorithm. So, for example if a query were made to find if User A were connected to User B then a BFS would allow us to find the shortest path and the route would be returned by the algorithm.

MIPS ISA Processor Design

This was a class project to implement a MIPS 32-bit ISA supported processor.

Single Cycle

The first implementation was to build a single cycle execution processor with access to elements in the execution cycle once per instruction per cycle. The ideal CPI of this processor was 1. It was modified to work for high latency RAM where the data and instruction hit's were stalled until the RAM let data into the memory controller. The datapath consisted of 32 word registers with an ALU and a program counter to increase the instruction count and fetch the next instruction.

5 stage Pipeline

As with every design, the first implemetation is the most naieve one. Using a 5 stage pipeline register model, instruction parallelism was achieved by having an instruction fetch, decode, arithmetic execution, memory and write back stage. The names of these stages clearly demonstrate their functionality. The important part was handling the hazards that were associated with instructions dependent on each other. These were solved using a forwarding unit. Load Word(lw) hazards were solved using stalls. And branch prediciton for single loop program execution was achieved.

Cache Integration and Dual Core Scale

To optimize for speed, a two way set associative cache with an LRU scheme was implemented to support instruction and data fetches quickly. This was integrated with a coherence controller that helped execute instructions on a dual core model. Data coherence was established using the MSI protocol. Finally Load Linked and Store Conditional operations were implemeted that allowed locking and unlocking mechanism for restricted access.

Stack

The entire design was coded in System Verilog and tested on DE-2 FPGA Altera board

Project Glass

Heads up wearable display

A successful attempt to recreate a cheaper version of existing market products that show information on a heads up display. This project was designed to display time, weather and news notifications as updated on an Android phone. The update events on the phone were subscribed by a Bluetooth Low Energy module(Microchip RN4020) hosted on an M0 Atmel SAMD microcontroller which together were a part of the heads up display's core functionality. This entire product along with the BLE, microcontroller all of which were hosted on a custom PCB costs around 25$ a piece and that is with respect to low production quantity.

Limitless Hardwood

A Boilermake IV 48 hour hackathon IOT project...

This particle module powered device renders user preference screens on two 0.96'' screens showing the most pertinent information precluding the usage of the phone for checking trivial but crucial details like date, time, temperature, stocks and others based on preference settings defined per device ID in a MongoDB framework. The events shown on the screen are subscribed to by the particle photon and the Node JS service satisfies the GET requests sent out by this very module. My Contributions:

  • The rendering on the screens is done via the SSD 1331 libraries using SPI interfacing.
  • It also shows bank account information received in a secure format using Capital One's API calls.
  • Packaging of the product is done keeping size and cost as a factor. The stand-alone equipment without any mass production costs $25 a piece. It could do wonders if mass produced.

DJarvis

A HackIllinois 2017 36 hour hackathon.

DJarvis - You're very own personal music assistant was made with the noble purpose of helping you cheer up or continue you're mood of relaxation. It integrates with the Microsoft Cognitive API to give the user a careful analysis of his/her feeling's and then alleivates, calms, or continues the current state of mind by playing music with respect to a parameter called valence which is integrated with the Spotify Web API. A valence parameter brought in using the Spotify Web API, which ranges from 0-1 helps convey the positivity of the song and compares with the Cognitive APIs detection of emotions of the person and this coherency helps achieve the ideal playlist whether the user is sad or happy. My Contributions:

  • Thought of the idea, and the product design and architecture.
  • Building the Node.js server side infrastructure.
  • Integrating webcam picture application using getUserMedia and displaying on a canvas and using Microsoft BlobStorage to save it to the cloud.
  • Using Microsoft Cognitive Emotions API to get the user clicked image's happiness level to send to server for Spotify song selection.
